Getting Started

Install and configure NeuroTrax AI in four simple steps — no extra dependencies needed.

Download NeuroTrax AI

Download NeuroTrax AI

Click the download button above to get the latest NeuroTrax_AI.exe installer. It's a self-contained package — no additional dependencies required.

Run it to auto-extract the folder

Double-click NeuroTrax_AI.exe to launch. It will automatically extract all necessary files into a local folder on your machine. Accept any Windows security prompts (Run Anyway) — this is a self-signed application.

After extraction, the folder will contain:

  • .env — configuration file (user, settings)
  • data/ — folder for all your JSON command & data files
  • AI.exe — the main NeuroTrax AI application

Open the extracted folder — you'll find the .env file, a data/ folder, and AI.exe. Configure the .env file first (see Step 3 below), then double-click AI.exe to start the AI.

🔒 Firewall prompt on first run: Windows will ask for network access — click Allow access for both public and private networks. NeuroTrax needs this for wake word detection and speech recognition.

Configure the .env file (do this before running AI.exe)

Open the extracted folder and find the .env file. Edit it with any text editor (e.g. Notepad) and set the user field to whatever name you'd like NeuroTrax to use — it can be anything you prefer, such as your first name or a nickname.

# .env — set your preferred name
user=charan
💡 This name is used for greetings and personalisation — it does not need to match your Windows account name.

Add your data to the data/ folder

The data/ folder holds all JSON files that teach NeuroTrax about your apps, files, and custom commands. You can populate it in two ways:

  • Run Data_maker.exe to automatically scan and index your system (apps, files, audio).
  • Manually edit commands.json, intents.json, and urls.json to add custom commands.

See the Data Structure section below for full details on each file.

Data Directory Structure

The data/ folder contains JSON files that the system loads at startup. These files define commands, intents, and indexed system paths.

File Purpose Source
commands.json Hardcoded command mappings (e.g. "exit" → "exit_program") Manual
intents.json Pattern-based commands with arguments (e.g. "search {query}") Manual
urls.json Maps site names to URLs (e.g. "google" → "https://google.com") Manual
search_url.json Search URL templates with {query} placeholder Manual
apps_list.json Maps app names to executable paths Manual
files_list.json Maps file names to file paths Generated
folders.json Maps folder names to directory paths Generated
audio_files.json Maps audio file names to file paths Generated
learned.json Stores user corrections and learned aliases Auto-AI

File Formats & Schema

1. commands.json

Defines direct mappings from a spoken phrase to a function name or a sequence of actions.

Simple Mapping

"shutdown neurotrax": "exit_program",
"play music": "play_songs"

Complex Mapping (Macros)

{
  "test mode": {
    "functions": ["play_song"],
    "process_commands": ["open youtube", "set volume to 50"]
  }
}

2. intents.json

Defines flexible commands with variable arguments using {variable} placeholders.

{
  "search {query} in youtube": {
    "function": "search_web",
    "args": ["query"]
  },
  "open {folder} in code": {
    "function": "open_in_code",
    "args": ["folder"]
  }
}

You can keep default values in intents. When using defaults the other variable should be null:

{
  "finds {query} in {provider}": {
    "function": "search_web",
    "args": {
      "query": null,
      "provider": "youtube"  // default
    }
  }
}

3. Generated Lists (apps_list.json etc.)

Simple key-value pairs: lowercase searchable name → absolute system path.

{
  "chrome": "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe",
  "notepad": "C:\\Windows\\System32\\notepad.exe"
}

4. urls.json — Simple URLs

{
  "chatgpt": "https://chatgpt.com",
  "youtube": "https://youtube.com"
}

5. search_url.json — Search Providers

Configure providers for the search_web function.

{
  "youtube": "https://www.youtube.com/results?search_query={query}",
  "google": "https://www.google.com/search?q={query}"
}

Data Generation

Double-click Data_maker.exe to automatically scan your system and generate the data files (files_list.json, folders.json, audio_files.json) in the /data folder.

💡 Run Data_maker.exe periodically to index newly installed apps and files.

Commands & Intents Reference

Overview of matching strategies the AI uses when a voice command is received.

Direct Match

Checks commands.json and learned.json for an exact phrase match. Instant execution.

Intent Match

Checks intents.json using regex / fuzzy matching to extract variable arguments (e.g., "set volume to 50%").

Fuzzy Fallback

Uses fuzzywuzzy to find the closest match across all indexed apps, files, and songs when no exact match exists.

Available Functions

These built-in functions can be referenced in commands.json and intents.json.

Function Arguments Description
set_volume value Sets the system volume level (0–100)
search_web query, provider Searches the web; default provider is youtube. Configure providers in search_url.json
open_in_code folder Resolves folder path from folders.json and opens it in VS Code
find_and_open_in_VS_code folder_name Searches for a folder from folders.json and opens it in VS Code
play_in_yt query Searches YouTube, selects the first result, and opens it in the browser
shutdown_computer countdown Asks for confirmation before shutdown; cancellable during the countdown
exit_program — Closes the NeuroTrax AI agent
play_songs — Plays local MP3 files from the indexed audio library

Data Loading & Execution Logic

Matching Priority

Exact Match

Checks commands.json and learned.json for a verbatim phrase match.

Intent Match

Checks intents.json using regex/fuzzy matching to extract arguments.

Fuzzy Fallback

Uses fuzzywuzzy to find the closest match across all indexed apps, files, and songs.

Execution

  • If a function name is found (e.g. exit_program), it is executed dynamically.
  • If a file path is found, the file is opened with its default application.
  • If a URL is found, it opens in the default browser.

Self-Learning

When you say "no, I mean...", the correction is saved to learned.json and used in all future sessions automatically.
